    • safe: <T, E>(
          generator: () => Generator<Result<unknown, E>, T, unknown>,
      ) => Result<T, E>

      Safely executes a generator function that yields Result values. Provides early exit on first error and automatic cleanup on failure. Similar to Rust's ? operator for chaining fallible operations.

      Example

      const result = advanced.safe(function* () {
        const user = yield getUser(id);        // If this fails, stops here
        const profile = yield getProfile(user.id); // Only runs if user succeeds
        const posts = yield getPosts(profile.id);  // Only runs if profile succeeds
        return { user, profile, posts };            // Only runs if all succeed
      });
    • safeAsync: <T, E>(
          generator: () => AsyncGenerator<Result<unknown, E>, T, unknown>,
      ) => Promise<Result<T, E>>

      Async version of safe() for generator functions that yield Promise.

      Example

      const result = await advanced.safeAsync(async function* () {
        const user = yield await fetchUser(id);
        const profile = yield await fetchProfile(user.id);
        return { user, profile };
      });
    • yieldFn: <T, E>(result: Result<T, E>) => Result<T, E>

      Helper function for yielding Results in safe() generators. Makes generator syntax cleaner by avoiding explicit yield syntax.

      Example

      const result = advanced.safe(function* () {
        const user = yield* advanced.yieldFn(getUser(id));
        return user;
      });
    • zip: <T, U, E>(resultA: Result<T, E>, resultB: Result<U, E>) => Result<[T, U], E>

      Combines two Results into a tuple Result. Fails if either input fails. Useful for operations that need both values to proceed.

      Example

      const nameResult = getUserName(id);
      const emailResult = getUserEmail(id);
      const combined = advanced.zip(nameResult, emailResult);
      // Returns: Ok([name, email]) or first Err encountered

      const user = match(combined, {
        Ok: ([name, email]) => ({ name, email }),
        Err: (error) => null
      });
    • apply: <T, U, E>(
          resultFn: Result<(value: T) => U, E>,
          resultValue: Result<T, E>,
      ) => Result<U, E>

      Applies a function wrapped in a Result to a value wrapped in a Result. This is the applicative functor pattern for Results.

      Example

      const addFn = ok((x: number) => (y: number) => x + y);
      const value = ok(5);
      const result = advanced.apply(addFn, value);
      // Returns: Ok(function that adds 5)

      // More practical example:
      const parseNumber = (s: string) => isNaN(+s) ? err("Not a number") : ok(+s);
      const add = ok((x: number) => (y: number) => x + y);
      const result = advanced.apply(
        advanced.apply(add, parseNumber("5")),
        parseNumber("3")
      );
      // Returns: Ok(8)
